# 機能設計

## 1. 動画視聴
- YouTube埋め込みで表示
- 科目 / 単元 / レッスンの階層で管理

## 2. 発問
- 目的：探究への入口（正解なし、メモへ誘導）
- 理解確認は小テストが担う（役割分担）
- 発問に答えた後、自然にメモ画面へ流れるUI設計
- **教師の作成フロー：**
  1. 動画を登録（科目・単元を指定）
  2. AIが発問を提案（編集可能な形式）
  3. 教師が編集
  4. DBへ登録

## 3. メモ機能
- **クローズド**：デフォルトは自分のみ閲覧可
- tiptapによるリッチテキスト編集
- 発問への回答から生まれた疑問・調べたことを記録する用途
- **タイムスタンプ連携**：
  - メモボタン押下時に現在の再生時間を自動記録（`player.getCurrentTime()`）
  - DBのメモテーブルに`timestamp_seconds`カラムで保存
  - タイムスタンプをクリックするとその時点から再生（`player.seekTo(秒数)`）

## 4. 共有投稿機能
- メモから「投稿する」を選択した時点でクラスに共有
- 匿名で共有される
- 「自分のメモ」と「他者のメモ」は明示的に区別して表示
- 共有メモは動画の下に一覧表示

## 5. 小テスト
- 役割：理解確認（正誤・採点）
- 内製化
- **問題形式：** 選択式 / 簡単な記述式 / 並び替え
- **採点機能：** あり
- **集計・可視化：** MVP対象外（将来検討）
