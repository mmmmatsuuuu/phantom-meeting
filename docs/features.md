# 機能設計

## 1. 動画視聴
- YouTube埋め込みで表示
- 科目 / 単元 / レッスンの階層で管理
- 科目ごとに Collapsible で折りたたみ可能（開閉状態を cookie で維持）

## 2. 発問
- 目的：探究への入口（正解なし、メモへ誘導）
- 理解確認は小テストが担う（役割分担）
- 発問に答えた後、自然にメモ画面へ流れるUI設計
- **教師の作成フロー：**
  1. 動画を登録（科目・単元を指定）
  2. AIが発問を提案（編集可能な形式）※MVP後検討
  3. 教師が編集
  4. DBへ登録

## 3. メモ機能
- **クローズド**：デフォルトは自分のみ閲覧可
- tiptapによるリッチテキスト編集
- 発問への回答から生まれた疑問・調べたことを記録する用途
- **タイムスタンプ連携**：
  - メモボタン押下時に現在の再生時間を自動記録（`player.getCurrentTime()`）
  - DBのメモテーブルに`timestamp_seconds`カラムで保存
  - タイムスタンプをクリックするとその時点から再生（`player.seekTo(秒数)`）
- **ツールバー**：Bold / Italic / Heading / BulletList / OrderedList / InlineCode / CodeBlock / Callout / Link / Table / （画像貼り付けは無効）
- シンタックスハイライト対応（`lowlight` + `@tiptap/extension-code-block-lowlight`）
- 表の挿入（グリッド選択UIで行・列を指定）

## 4. 共有投稿機能
- メモから「投稿する」を選択した時点でクラスに共有
- 匿名で共有される
- 「自分のメモ」と「他者のメモ」は明示的に区別して表示
- 共有メモは動画の下に一覧表示
- Supabase Realtime でリアルタイム更新

## 5. 小テスト
- 役割：理解確認（正誤・採点）
- **問題形式：** 選択式 / 簡単な記述式 / 並び替え
- **採点機能：** 選択式・並び替えは自動採点（サーバーサイド）、記述式は模範解答表示による自己採点
- 提出後に解説を表示（正解・不正解関係なし）
- 並び替え問題は @dnd-kit によるドラッグ&ドロップで回答
- **小テスト完了ゲート：** 小テストを1回でも提出するまで他の生徒の共有投稿を非表示にする
  - 小テストなし → 最初から投稿一覧を表示
  - 小テストあり・未完了 → 投稿一覧をロック表示
  - 小テストあり・完了済み → 表示
- **回答詳細の保存：** 提出時に各問の回答内容・正誤を `quiz_attempt_answers` に保存
  - 選択式・並び替え：`is_correct = true/false`
  - 記述式：`is_correct = null`（自己採点のため）
- **JSONインポート：** Claude等のAIが生成したJSON形式の問題を貼り付けて一括インポート可能

## 6. ロール管理（admin 機能）
- admin が student を teacher に昇格させる
- `profiles.role` / `is_approved` を変更する専用ページ（`/admin/users`）
- admin は DB に直接設定（開発者自身）

## 7. Tiptap エディタリッチ化
- ツールバー追加（Bold / Italic / Heading / BulletList / OrderedList / InlineCode / CodeBlock / Callout / Link / Table）
- シンタックスハイライト（`lowlight` + `@tiptap/extension-code-block-lowlight`）
- 表の挿入：グリッド選択UIで行・列を指定して挿入、行・列の追加・削除に対応
- 画像の貼り付け（小テスト作成画面のみ有効）：
  - コピペした画像を自動で ImageKit にアップロードし URLで保存・表示
  - メモエリアでは画像の貼り付けを無効化（ペーストで破棄）
  - 挿入後の画像はドラッグで幅をリサイズ可能
  - `RichContent`（読み取り専用）は画像の表示のみ対応

## 8. データ出力
- **生徒向け（メモ一覧）：**
  - `/memos` ページで全レッスンのメモを科目→単元→レッスン階層で一覧表示
  - Markdown ファイルとしてダウンロード（授業単位 / 単元単位 / 科目単位）
  - サイドバーに目次（IntersectionObserver によるアクティブハイライト）
  - 各授業に YouTube 動画リンクを表示
- **教師向け：** レッスンごとの投稿・小テスト回答データを CSV でエクスポート（将来検討）

## 9. 画面遷移フィードバック
- Next.js App Router のデータ取得に起因するタイムラグを補う
- リンク・カードをクリックした瞬間に画面上部にトップローディングバーを表示
- 実装：`nextjs-toploader` を `app/layout.tsx` に一度追加するだけで全画面に適用
- フォーム送信など明示的なアクションは既存の toast フィードバックで対応

## 10. プロフィール編集（生徒向け）
- `/profile` ページで学籍番号・備考を編集可能
- 表示名は編集不可（Google OAuth から取得した名前を使用）
- 保存成功 / 失敗を toast で通知

## 11. 生徒一覧（教師向け）
- `/teacher/students` ページで全生徒の学籍番号・表示名・ロール・備考を一覧表示
- 表示名・学籍番号で検索、学籍番号・表示名でソート
- 学籍番号・備考をインライン編集

## 12. 生徒メモ閲覧（教師向け）
- `/teacher/lessons/[lessonId]/memos` ページで特定レッスンの生徒メモを閲覧
- **2段階ロード × クラス絞り込み**で通信量と UX を両立
  - 学籍番号の桁構造（1桁目=学年・2桁目=クラス・3〜4桁目=出席番号）を利用して絞り込み
  - 第1段階：学年・クラスを選択 → 約40人のメタデータ（表示名・学籍番号・メモ件数）を取得
  - 第2段階：生徒行をクリック → その生徒のメモをオンデマンドで取得・アコーディオン展開
- `/teacher/contents` のレッスン行からリンク

## 13. 生徒ダッシュボード（ホーム・レッスン内接点）
- **目的**：メモ一覧・小テスト結果への導線を強化し、生徒が「自分の今の状況」を一目で把握できるようにする
- **ホーム（授業外・家庭学習の接点）**：
  - 学習状況サマリー4カード（小テスト受験数 / 平均得点率 / 要復習数 / メモ数）。カードクリックで `/quiz-results`・`/memos` へ
  - 「つぎにやること」：要復習（最新得点率60%未満）レッスンを優先提示、次いで未受験レッスン
  - レッスン一覧の各行にステータスバッジ（✓得点率 / ⚠要復習 / 未受験・メモ件数）
- **レッスンページ（授業中の接点）**：
  - タイトル下に「あなたの状況」ステータスバー（直近得点率・受験回数・メモ件数）
  - 小テスト提出直後に要復習ナッジ（ほかに要復習レッスンが n 件 → 小テスト結果へ）
- 全ロールに表示する（教師・管理者が「生徒にどう見えるか」を自分のデータで確認できるようにするため）
- 集計クエリは quiz_attempts / memos 各1回に抑え、無料枠に配慮

## 14. 教師ハブ（/teacher）
- 教師機能の起点となるトップページ。カード導線を2グループで整理
  - **授業をつくる**：コンテンツ管理（レッスン登録・小テスト作成もここから）
  - **生徒をみる**：小テスト分析 / 生徒一覧 / データエクスポート
- ドロップダウンメニューも同じ2グループ構成（先頭に「🏫 教師ページ」リンク）
- 「レッスンを登録」単独メニューは廃止し、コンテンツ管理の単元行「📺 レッスン追加」に一本化

## 15. 分析（/teacher/analytics）
- 小テスト結果の分析をタブで切り替える統合ページ（旧 `/teacher/quiz-analytics` はリダイレクト）
- **単元別タブ**：学年・クラス・科目を選択 → 授業×設問の平均正答率ヒートマップ
- **レッスン別タブ**：レッスンを選択 → 生徒×設問の回答一覧テーブル
  - 各生徒の行に得点・設問ごとの正誤（正解=○、誤答=選んだ内容を表示、ホバーで正解確認）
  - 記述式は記入内容をそのまま表示（ホバーで全文）
  - メモ列に生徒ごとのメモ件数を表示
  - サマリーに対象生徒数・受験済み人数・メモ記入人数と生徒メモ閲覧ページへの導線。生徒名クリックで個人詳細へ
- 集計は各生徒の**最新受験のみ**を対象とする（エクスポートと同じルール）
- **生徒別タブ**：行=生徒・列=レッスンのヒートマップ（最新得点率）。生徒名クリックで個人詳細へ

## 16. 生徒個人詳細（/teacher/students/[userId]）
- 特定の生徒の学習状況を授業全体・単元・レッスンの3階層で確認できる教師向けページ
- 全体サマリー（受験数 / 平均得点率 / 要復習数 / メモ数）
- 単元ごとに平均得点率バッジ＋レッスン別テーブル（最新得点率・受験回数・メモ件数）
- 導線：生徒一覧の表示名リンク / 生徒別ヒートマップの生徒名リンク
- 集計は生徒ダッシュボードと同じ `buildStudentDashboard` を再利用（クエリは profiles / contents / quizzes / quiz_attempts / memos 各1回）

## 17. いいね機能（見送り）
- 検討の結果、現時点では導入しない方針
- 見送り理由：SNS的な歪み（受けを狙った投稿・いいね数への過剰な意識）のリスク
- 代替として Phase 17 の匿名プライベートコメント機能で「共有が意味を持った感覚」を実現予定
